1 Problem Definition

금융 서비스에서 사용자가 업로드하는 문서는 블러, 크롭, 글레어 등으로 불완전합니다.

핵심 과제: input quality failure(재촬영 필요)와 semantic uncertainty(담당자 확인 필요)를 구분하는 것. retake 남용은 UX 저하, review 남용은 운영 비용 증가.

Input Types

신분증(주민등록증)과 통장사본. 문서 유형별 필수 필드를 정의하고 추출.

Decision Space

PASS 자동 승인   RETAKE 재촬영
REVIEW 담당자 확인   INVALID 문서 불일치

Input Conditions

blur, crop, glare, downscale, compression, rotation, low contrast 등 9가지 열화 조건.

2 4-Gate Pipeline

이미지 입력부터 최종 판정까지 4단계 Gate를 순차 통과합니다.

Gate 1

입력 유효성 검증

이미지 읽기, 최소 크기(100px), 검은/흰 화면 비율 검사. 실패 시 OCR 스킵. blur·저해상도는 기록만 하고 통과.

retake 다음 Gate
OCR Extraction

PaddleOCR 텍스트 추출

전체 텍스트 추출 → 문서 유형별 규칙으로 구조화 필드 파싱. CTC decoder monkey-patch로 글자별 confidence 추출.

Gate 2

문서 유형 검증

키워드("주민등록증", "통장", "은행" 등)로 유형 판별. 명확 불일치 → invalid. 애매 + 품질 문제 → retake. 애매 + 품질 정상 → VLM(Qwen3-VL) 분류.

invalid retake 다음 Gate
Gate 3

필수 정보 존재 검증

신분증: name, id_number, address, issue_date / 통장: name, account_number, bank_name. 일부 문제 시 VLM reread 보완 시도.

retake review 다음 Gate
Gate 4

형식 검증 + Confidence

필드별 confidence 임계값(핵심 0.7, 이름 0.6, 보조 0.5) + 글자별 confidence(0.5 미만 지적) + 주민번호 형식 검증.

review pass

5 VLM Reread Strategy

OCR 실패/저신뢰 필드에 대해 VLM을 보조적으로 활용하는 보완 전략

Field TypeSituationVLM ActionResult
Text (name, bank_name, address, issue_date) Missing VLM 값으로 보완 (conf 0.8) pass 가능
Text Low confidence 일치 시 승격, 불일치 시 VLM 값으로 대체 (conf 0.6) pass 가능
Numeric (id_number, account_number) Missing VLM 값 넣되 low conf 유지 review 유지
Numeric Low confidence OCR-VLM 정확 일치 시에만 승격 불일치 시 review
핵심 원칙: VLM에게 "불확실하면 unknown" 강제. 숫자 필드는 OCR-VLM 정확 일치 시에만 승격. 틀린 값의 자신감이 가장 위험하다.

7 Design Principles

시스템 설계의 핵심 원칙

🎯

Decision under Imperfect Input

"이 입력으로 판정이 가능한가?"를 묻습니다. 입력 품질 실패(retake)와 의미적 불확실성(review)을 명확히 분리합니다.

🛡️

Conservative Numeric, Flexible Text

주민번호·계좌번호는 OCR-VLM 정확 일치 시에만 pass. 이름·은행명은 VLM 보완으로 유연하게 처리합니다.

🔗

VLM as Secondary Signal

VLM은 단독 판정자가 아닌 보조 신호. invalid_doc_type 판정에는 반대 타입의 strong OCR signal이 반드시 필요합니다.

⚡

Fast Path First

OCR 결과를 SSE로 즉시 전송, VLM은 필요 시에만 후속 호출. 정상 문서는 VLM 없이 빠르게 pass 판정.

🚪

Early Exit Gates

Gate 1 실패 시 OCR 스킵. 각 Gate에서 결정이 나면 후속 단계를 실행하지 않아 불필요한 연산을 방지합니다.

🔬

Character-level Confidence

라인 평균이 높아도 특정 글자 confidence가 낮으면 review. CTC decoder monkey-patch로 글자별 신뢰도를 추출합니다.

10 Known Limitations

MVP의 한계와 개선 방향

이름 추출 Heuristic

bbox 없이 텍스트 순서 기반 추출. OCR line order가 깨지면 오인식 가능.
→ bbox 좌표 기반 후보 선택, 문서 타입별 필드 위치 템플릿.

은행명 매칭

하드코딩된 은행명 리스트와 exact match. OCR 오인식 시 실패.
→ fuzzy matching, 은행명 리스트 외부 설정 분리.

Glare 감지 비활성

흰색 배경과 실제 glare 구분 불가로 비활성화 상태.
→ gradient 기반 국소 밝기 변화 분석, 학습 기반 classifier.

향후 확장

Rule-based → Learned decision model, adaptive threshold, 다국어 OCR, A/B testing framework.
